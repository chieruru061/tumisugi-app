# アプリケーション要件定義

## 1. 概要
積みゲーを管理できるアプリケーション。  
ASP.NET Core MVCの学習の成果物でもある。

## 2. 機能要件
- データの表示機能:

    - リンクにアクセスすると、当機能のお積みもの一覧画面が表示
    - 検索機能がある。
    - 10件ごとに表示される。ページャーを設置する。 
    - ソート機能があり、日付やタイトルでソートができる。

- データの編集機能: 専用画面を用意

    - ゲームの名前やジャンル、状態等を編集できる。
    - スクロールをせず一画面ですべて入力できる。
- データの削除機能: 専用画面を用意

    - データを物理削除できる。
- データの作成機能: 専用画面を用意

    - ゲームの名前やジャンル、状態等を入力できる。
    - スクロールをせず一画面ですべて入力できる。

## 3. 非機能要件
- スケーラビリティ: 将来的にアニメや本の管理もできるように構成すること。
- パフォーマンス: 100件程度のデータでも快適に動作すること。
- セキュリティ: 
    - ローカル環境での利用を前提とし、認証機能は実装しない。
    - ページの遷移はフレームワークのルーティング機能を利用する。
- 保守性: コードの可読性を重視し、拡張しやすい構成とする。

    [ASP.NET Core公式ドキュメント](https://learn.microsoft.com/ja-jp/aspnet/core/?view=aspnetcore-9.0)の推奨に準拠する。

## 4. 技術スタック
- フレームワーク: ASP.NET Core MVC, Entity Framework
- データベース: SQLite
- Webサーバー: Kestrel(ローカルホスト)
- フロントエンド: ASP.NET Core MVC(Razor)

## 5. やらないこと
### 前提として、アプリ作成後に機能を追加する可能性がある
- ユーザ管理機能やログイン機能　　
    
    主にルーティングやアクションメソッドの動きについての学習成果をしたいため。

- APIやゲームのストアページの外部リンク

    今回の学習に沿わないため。ストアページ等はユーザに入力させるのは苦痛かつ、メモ領域を自由に使っていただく設計のため。
